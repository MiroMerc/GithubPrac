GenAI Prompting Results Prompt used (Plan-Based): > “I have a dataset of passenger traffic for six airports (ATL, FRA, PKX, ORD, JFK, DEN) from 2020-2025. Following my plan: 1. Create a tribble in R. 2. Pivot the data wider for a table. 3. Plot the trends using ggplot2 with comma-formatted labels. 4. Hide the code in the final output but ensure it is functional.”

Result: The tool (Gemini) provided the R code used in Section 1 of this report. It correctly interpreted the pivot_wider requirement and used scales for the formatting.

Comparison of Plan-Based vs. Generic Prompting Generic Prompt Result : The generic prompt (“Analyze this airport data in R”) produced a basic plot without proper formatting like label_comma or organized “Small Multiples,” often leaving the data in a long format that is harder to read in a summary table. It also produced broken code, that took multiple prompts to repair

Comparison Narrative: The plan-based prompt resulted in a much more “presentation-ready” document. By specifically naming the functions (pivot_wider, label_comma) and the structure (Table vs. Plot), the output required zero manual fixing. In contrast, the generic prompt output usually requires multiple follow-up questions to fix axis labels, handle the data structure, or hide the code chunks.

The primary difference is the output, plan-based approach ensures the AI understands the intended final product, on the other hand the generic prompt doesn’t understand fully, creating a less professional and complete document

Code Appendix Monte Carlo

f_mc <- function(x) -(x - 4)^2 + 4

run_sim <- function(n) {
  tibble(
    x = runif(n, 2, 6),
    y = runif(n, 0, 4),
    n_label = paste0("n = ", n)
  ) |> 
    mutate(status = ifelse(y <= f_mc(x), "on/below", "above"))
}

mc_results <- bind_rows(run_sim(10), run_sim(100), run_sim(1000), run_sim(10000))

ggplot() +
  geom_function(fun = f_mc, xlim = c(2, 6), color = "black", linewidth = 1) +
  geom_point(data = mc_results, aes(x, y, color = status), alpha = 0.5, size = 0.7) +
  facet_wrap(~fct_relevel(n_label, "n = 10", "n = 100", "n = 1000"), ncol = 4) +
  scale_color_manual(values = c("above" = "firebrick", "on/below" = "forestgreen")) +
  theme_minimal()
